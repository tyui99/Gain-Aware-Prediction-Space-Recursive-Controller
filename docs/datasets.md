# Dataset Preparation

This package does not redistribute the datasets. Download them from the original sources and reorganize them into the layouts below.

## Download Links

- Kvasir-SEG: https://datasets.simula.no/kvasir-seg/
- CVC-ClinicDB: https://www.kaggle.com/datasets/balraj98/cvcclinicdb
- CVC-ColonDB: https://www.kaggle.com/datasets/longvil/cvc-colondb
- PolypGen: https://www.kaggle.com/datasets/debeshjha1/polypgen-video-sequence

## Training Dataset Layout

Training uses only Kvasir-SEG.

```text
data/light/kvasirseg/
├── train/
│   ├── images/
│   └── masks/
└── val/
    ├── images/
    └── masks/
```

Filenames under `images/` and `masks/` must match one-to-one.

If you already use a seed-specific split folder generated by your own preprocessing code, the runtime loader also accepts `seed_x/train|val`, but the public-facing layout above is sufficient.

## Evaluation Dataset Layout

The evaluation helpers accept either the train/val layout above or a flat paired directory for transfer datasets:

```text
data/light/eval/ClinicDB/
├── images/
└── masks/

data/light/eval/ColonDB/
├── images/
└── masks/

data/light/eval/PolypGen/
├── images/
└── masks/
```

Use `--eval-only` when `--data-root` points to a flat paired directory instead of a train/val split.

## Notes

- Kvasir-SEG is the only training dataset used by the paper.
- ClinicDB, ColonDB, and PolypGen are used only for transfer evaluation in this package.
- The loader resizes images and masks to `352 x 352` at runtime.
- Follow the original dataset licenses and usage restrictions before redistribution or public mirroring.
